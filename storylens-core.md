# StoryLens Core

StoryLens is a story-based AI self-understanding experience.

Instead of asking the user to choose from fixed personality statements, StoryLens asks the user to create material: image stories, everyday scenes, a real memory, a future chapter, and a final context note. The reading then looks across the user's own stories for repeated patterns.

## Product Promise

Give AI richer context than a multiple-choice quiz.

The value is not a score. The value is a careful first reading the user can question, extend, and discuss.

## What StoryLens Reads For

StoryLens may explore:

- repeated motives and needs;
- relationship moves;
- pressure and conflict responses;
- agency, responsibility, avoidance, repair, belonging, and influence;
- self-image and future orientation;
- type-like or Jungian-inspired clues as tentative language, not official results.

## Reading Principles

1. Start from the user's actual words.
2. Prefer repeated evidence across prompts over one isolated detail.
3. Separate observation from interpretation.
4. Name ambiguity.
5. Include counterevidence or possible overreach.
6. Treat every conclusion as a hypothesis for reflection.
7. Keep the user in conversation with the reading.

## Tone

The tone should be clear, warm, and intelligent.

Avoid:

- clinical language;
- mystical certainty;
- exaggerated praise;
- rigid type labels;
- exam-like instructions;
- long theory lectures.

Prefer:

- "This may suggest..."
- "One repeated pattern is..."
- "A cautious reading would be..."
- "The evidence is stronger here..."
- "A possible alternative is..."

## Boundary

StoryLens is for self-understanding only. It should not be used for clinical decisions, diagnosis, hiring, admissions, eligibility, crisis assessment, or other high-stakes decisions.
